# Publishing

Use this reference only for logged-in Xiaohongshu publishing work.

## Hard constraints

- Google Chrome only
- Chrome DevTools MCP attach only
- no relay
- no PinchTab for logged-in actions
- v0.1 does not auto-click final publish

## v0.1 publishing boundary

This skill prepares a publish-ready package and preflight checklist.

It does not take ownership of the final publish click in v0.1.

That means:

- title ready
- cards ready
- CTA ready
- checklist ready
- Chrome attach route ready
- final click still human-confirmed

## Pre-publish order

1. title passes length check
2. cards exist in the expected count
3. CTA is present
4. post type and layout match the package
5. Chrome session is available
6. operator confirms the post is visually acceptable

## Manual publish checklist items

The operator should still confirm:

- final title in the composer
- cover choice
- body paste result
- image upload order
- visibility/options
- final publish click

## Preview-first mindset

Treat publishing like:

- prepare
- preview
- verify
- confirm

not:

- generate
- blindly send

## Post-publish handoff

After publish, capture:

- final title
- final link
- what actually went out
- anything that drifted from the package
- one next-step improvement
