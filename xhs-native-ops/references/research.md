# Research

Use this reference when a topic exists only as a rough signal and still needs to
be shaped into something Xiaohongshu can carry.

## Goal

Turn messy upstream signal into a direction that can support:

- one clear hook
- one clear angle
- one page-level card structure

Research is not the same thing as drafting. Stop research when you can answer:

1. what feels fresh here
2. what Xiaohongshu already rewards around this topic
3. what we can say credibly

## Signal sources

### X

Use X as the frontier signal source:

- new tools
- new workflows
- new opinions
- new emotional angles
- new cases before they become over-translated

Capture:

- post link
- author
- key claim
- why it spread
- what feels new

### Xiaohongshu

Use Xiaohongshu as the packaging and comment signal source:

- title pattern
- first-screen pattern
- save-heavy vs comment-heavy content type
- repeated user question
- whether the post wins because it is actually useful or only visually strong

Capture:

- post link
- title pattern
- cover pattern
- opening structure
- repeated comment question

## Minimum topic fields

Record at least:

- `topic`
- `freshness`
- `source_platform`
- `signal_type`
- `x_signal`
- `xhs_pattern`
- `repeated_question`
- `proof_source`
- `risks`
- `suggested_angle`

## Compatibility with upstream research outputs

If local research outputs already exist, they may be used as upstream
material. For example:

- keyword radar summaries
- viral sample collections
- comment extraction summaries
- topic pool notes

These are optional inputs only. This skill must remain usable without them.

## Good stopping point

Research is good enough for this skill when you can write:

- one working hook
- one clear angle
- one card outline

If you still have five competing directions, you are still in research, not in
topic design.
