# Visual Packaging

Use this reference before generating or reviewing cards.

## Goal

Produce cards that feel:

- native to Xiaohongshu
- easy to scan
- calm and clear

Do not optimize for "maximum information density."

## Page roles

### Cover page

The cover should do three things:

1. establish the topic
2. create curiosity or clarity
3. stay readable at a glance

Cover rules:

- short headline
- one support line maximum
- strong hierarchy
- no wall of text

### Content page

Each content page should hold one idea only.

Content rules:

- one page = one point
- short title
- 1-3 short lines of body text
- visible whitespace

## Layout rules

### Six-card package

- 1 cover
- 5 content cards

Best for:

- compact arguments
- simple method posts
- strong opinion posts

### Nine-card package

- 1 cover
- 8 content cards

Best for:

- deeper methods
- more layered breakdowns
- save-oriented practical posts

## Density rules

These are v0.1 defaults:

- cover headline: target <= 18 characters
- cover support line: target <= 24 characters
- content title: target <= 16 characters
- content body: target <= 60 characters

If a page exceeds these limits, treat it as:

- a warning first
- a revision blocker if the page becomes hard to scan

## Native-feel rules

The cards should not look like:

- a pitch deck
- a technical infographic
- a programming dashboard

They should feel closer to:

- a clean creator note
- a soft information card
- a platform-friendly content slide

## Template intent

v0.1 uses:

- one cover template
- one six-card content template
- one nine-card content template

This is enough for a minimum closed loop. Template variety can come later.
