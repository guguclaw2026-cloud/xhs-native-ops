# Review Rubric

Use this reference to decide whether a package is ready, needs revision, or
should be held.

## Ready

Mark as `ready` only when:

- the hook is strong
- the angle is singular
- the card outline fits the chosen layout
- title length is under control
- cards are readable
- the CTA is clear
- the package feels native enough for Xiaohongshu
- there are no meaningful packaging mismatches left to fix

## Revise

Mark as `revise` when:

- the idea is usable
- but the packaging is still weak

Typical revise cases:

- title too long
- cover too dense
- card titles too vague
- body text too heavy
- CTA exists but feels weak
- the package is usable, but the score is inflated by generic goals or broad claims
- the outline does not land cleanly on the chosen layout

## Hold

Mark as `hold` when:

- the topic is still unclear
- the angle is split
- the content feels off-platform
- the post would likely hurt fit more than help growth

## Native-feel check

Ask these questions:

1. Does the cover stop the scroll?
2. Can each page be understood quickly?
3. Does the package feel like Xiaohongshu content, not a system export?
4. Is the post likely to earn saves, comments, or better-fit attention?

If too many answers are "no", it is not ready.

## Blocking issues

These should block publishing:

- missing required package sections
- title clearly over limit
- too few cards for the chosen layout
- unreadable cover
- no CTA
- broken card generation

## Non-blocking issues

These can survive into later revisions:

- template style could be prettier
- wording is good but not yet sharp
- comments angle could be stronger
- support line could be refined
